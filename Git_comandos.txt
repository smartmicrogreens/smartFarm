Comandos de Git comunes usados en varias situaciones:

****** Más info en : https://git-scm.com/docs

Comenzar un area de trabajo (ver: git help tutorial)
   clone      Clona (copia) un repositorio en una nueva carpeta.
   init       Crea un repositorio Git vacío o reinicializa uno existente.

Trabajar en el cambio actual (ver: git help everyday)
   add        Agrega los contenidos de un archivo al índice.
   mv         Mueve o renombra un archivo, carpeta o enlace.
   reset      Resetea el HEAD actual al estado especificado.
   rm         Remueve archivos del árbil de trabajo y del índice.

Examinar la historia y estado (ver: git help revisions)
   bisect     Usa búsqueda binaria para encontrar el commit que introdujo un bug en específico.
   grep       Muestra lineas que coinciden con un patrón.
   log        Muestra la lista de commits realizados.
   status     Mostrar el árbol de trabajo actual -> Muestra consejos de qué hacer.

Modificar la historia en común
   branch     Listar, crear, eliminar ramas del proyecto.
   checkout   Cambia entre ramas o restaura árboles de archivos de trabajo (working tree files).
   commit     Notifica de cambios en el repositorio -> Crea nueva "versión".
   diff       Muestra cambios entre commits y el árbol de trabajo, etc.
   merge      Unir dos o más historias de desarrollo juntas.

Colaborar (ver: git help workflows)
   fetch      Descargar objetos y referencias de otro repositorio.
   pull       Hace un "fetch" y un "merge" del repositorio actual con otro repositorio o una rama local.
   push       Actualiza repositorio remoto con modificaciones realizadas.
